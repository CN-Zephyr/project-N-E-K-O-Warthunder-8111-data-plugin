## 2.1 宿主运行时注册注意

当前工作区里 `N.E.K.O\plugin\plugins\neko_warthunder` 必须是指向独立插件仓库的 junction。手动启动宿主时不要额外设置 `PLUGIN_CONFIG_ROOT` 指向外层工作区；否则宿主可能同时扫到 junction 和独立仓库目录，出现重复插件 ID 或加载旧副本。建议使用：

```powershell
cd D:\Users\zheng\Documents\Code\N-E-K-O-Warthunder\N.E.K.O
uv run python launcher.py
```

启动后确认注册：

```powershell
Invoke-RestMethod http://127.0.0.1:48916/plugins
Invoke-RestMethod -Method Post http://127.0.0.1:48916/plugins/refresh
Invoke-RestMethod -Method Post http://127.0.0.1:48916/plugin/neko_warthunder/start
```

若 refresh 报出外层独立仓库目录名的重复/无效项、出现 `neko_warthunder_1`，或 `test_say` 在 `dry_run=true` 下仍返回 `pushed=true`，不要继续测试；先修正 junction / 启动环境，确保宿主加载的是独立仓库最新代码。
